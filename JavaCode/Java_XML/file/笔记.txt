1. 解析xml的方式通常有4种 DOM SAX DOM4J JDOM

2.各种解析方式的相关问题
	1)乱码问题,通过xml的报文头确认
	
3.4种解析方式的区别
	1) dom会把xml全部放到内存中,在内存中形成一个dom树.
	优点:结构清晰,代码容易编写, 由于加载在内存中,可以进行修改.
	缺点:如果xml文件比较大,对内存要求很高
	2) sax解析,是基于事件的解析,一条一条语句的去解析,根据handler执行不同的方法 
	startDocument-->startElement-->endElement-->entDocument
	优点:采用的模式是事件驱动,消耗内存小对于较小
	缺点:只能顺序访问.
	3)JDOM
	优点:仅使用具体类而不是用接口
	4)DOM4J
	优点:性能优异,灵活性好,功能强大,易使用
